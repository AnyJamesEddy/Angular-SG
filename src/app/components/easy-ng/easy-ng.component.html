TypeScript

  @Component({...})
  export class EasyNgComponent {
  name: string = 'Eddy';
  }

HTML

  <p style="font-style: italic">
      "Hello! I'm  { {  name  } } and this is a small guide for noobs like me!"
  </p>

I've linked the TS variable "name" to the HTML template using a double bracket. This is called string interpolation and it basically allows to render string or number variables created on the TS file linked to the template. It is one of the three main ways to make your Typescript code work with your HTML template!

These communication methods are:
  • String Interpolation
  • Property Binding
  • Event Binding

    If you click on Button 1, it will become green, while Button 2 will become red and viceversa. This has been made via event binding.

    Here's the snippets:

    TypeScript

      @Component({...})
  export class EasyNgComponent {
  name: string = 'Eddy';
  }
   b1Pressed: boolean = false;
   b2Pressed: boolean = true;

  onClick1() {
  if (this.b1Pressed === false) {
      this.b1Pressed = !this.b1Pressed;
      this.b2Pressed = !this.b2Pressed;
    }
  }

   onClick2() {
  if (this.b2Pressed === false) {
      this.b1Pressed = !this.b1Pressed;
      this.b2Pressed = !this.b2Pressed;
    }
  }

    Using event binding, it is possible to link some feature (like a click!) of an element to a user coded action, like the color switch you can see above.

    Also the style properties, those funny instructions that make the button look so cool, changed with the event. In this example, they are distributed in a group of class members, but bound as a single property using "ngClass", a directive which allows to "append" one or more classes to the element class group.
